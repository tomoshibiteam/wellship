## Phase 3: 実船PoC対応 ⏳

**目標**: 実際の船舶環境での運用検証

### 3.1 技術要件

> 💡 インタビュー結果: 内航船では1日を通して完全オフラインになることはない

- [ ] 低品質ネットワーク対応（低帯域・不安定な接続への最適化）
- [ ] リクエストのリトライ・タイムアウト設定
- [ ] 軽量なデータ転送（画像圧縮、APIレスポンス最小化）
- [ ] ローカルキャッシュ活用（頻繁にアクセスするデータ）
- [ ] タブレット端末最適化

### 3.2 運用要件

- [ ] NFC/QRカード発行フロー
- [ ] 船員登録プロセス
- [ ] 日次運用マニュアル
- [ ] トラブルシューティングガイド

### 3.3 データ収集

- [ ] 使用ログ収集機能
- [ ] 滞在時間・離脱率計測
- [ ] エラー発生率モニタリング

### 3.4 PoC評価基準

| 指標 | 目標値 |
|------|--------|
| フィードバック回収率 | 80%以上 |
| 平均入力時間 | 30秒以内 |
| システム稼働率 | 99%以上 |
| ユーザー満足度 | 4.0/5.0以上 |
