## エグゼクティブサマリー

**WELLSHIP**は、内航海運向けの**食材調達プラットフォーム**です。

従来の「献立管理SaaS」から**「食材流通を担う商社プラットフォーム」**へ転換しました。

### 新しい価値提案
- 🆓 **SaaSは無料**: 献立管理・栄養計算を無料で提供
- 🛒 **ワンクリック発注**: 献立から自動で食材を発注（Checkout）
- 💰 **予算内最適化**: 1人1日1,400円の予算内で最適な食材を提案
- 🎁 **Boost食材**: ロス食材をお得に提供し、フードロス削減に貢献

---

## ピボット戦略

### ビジネスモデルの転換

| 項目 | Old (SaaSモデル) | New (商社モデル) |
|------|------------------|------------------|
| **主な価値** | 献立作成ツール | 食材発注プラットフォーム |
| **収益源** | サブスクリプション料金 | 食材流通マージン |
| **SaaS料金** | 月額50,000円/船舶 | **無料** |
| **ユーザー獲得** | 営業コスト大 | 無料で拡大→商流へ誘導 |

### 転換の理由
1. **SaaS料金への抵抗**: 業界のデジタル化遅れにより、ツール費用への支払い意欲が低い
2. **商流の規模**: 食材調達費用は船員の人件費に次ぐ大きなコスト
3. **Boost食材の可能性**: フードロス問題への貢献と原価低減の両立

## ビジネスモデル

### 収益構造

```
┌─────────────────────────────────────────────────────────┐
│                    ユーザー（船舶）                       │
│  献立作成 → 調達リスト生成 → 発注確定（Checkout）         │
└──────────────────────┬──────────────────────────────────┘
                       │ 発注データ
                       ▼
┌─────────────────────────────────────────────────────────┐
│                    WELLSHIP                              │
│  発注集約 → 仕入先マッチング → 配送手配                   │
│  マージン: 10-15%                                        │
└──────────────────────┬──────────────────────────────────┘
                       │ 仕入れ
                       ▼
┌─────────────────────────────────────────────────────────┐
│                    仕入先（Supplier）                    │
│  食材卸売業者、生産者、ロス食材提供元など                 │
└─────────────────────────────────────────────────────────┘
```

### 予算モデル
- **1人1日予算**: 1,400円（業界標準）
- **最低消化率**: 90%（予算を適切に使い切ること）
- **Boost食材（ロス品）**: 通常価格の30-50%オフで提供

### 収益予測（3年間）
- **Year 1**: 100船舶 × ¥1,400/日 × 30日 × 20人 × 10% = **¥84,000,000**
- **Year 2**: 300船舶 → **¥252,000,000**
- **Year 3**: 800船舶 → **¥672,000,000**

## 製品・サービス

### 1. 献立管理（無料SaaS）

#### Manager機能
- 📊 ダッシュボード: 予算消化状況の可視化
- 📅 献立承認: AI生成献立の確認・承認
- 📈 分析: 栄養バランス、コスト分析

#### Chef機能
- 🤖 AI献立生成: 栄養・予算・在庫を考慮
- 📋 日別メニュー表示
- 💬 フィードバック送信

### 2. 調達・発注（Checkout機能）✨NEW

- 📦 **調達リスト自動生成**: 献立から必要食材を自動計算
- 💰 **予算消化プログレスバー**: リアルタイムで予算状況を表示
- 🎁 **Boost食材バッジ**: お得なロス食材を視覚的に強調
- 🛒 **ワンクリック発注**: 「発注する」ボタンでOrderを作成
- 📄 **発注履歴**: 過去の発注を確認・追跡

### 3. Boost食材プログラム

- フードロス削減に貢献
- 仕入先からロス品を安価で調達
- ユーザーには「Special Offer」として提案
- Win-Win-Win（ユーザー/WELLSHIP/仕入先/環境）

## データモデル

### 新規追加モデル（商社機能）

#### Vessel拡張
```prisma
model Vessel {
  budgetPerDay Int @default(1400) // 1人1日あたりの予算（円）
  orders       Order[]            // 発注履歴
}
```

#### Ingredient拡張
```prisma
model Ingredient {
  price       Int     @default(0)    // 標準単価（円）
  isBonus     Boolean @default(false) // Boost食材フラグ
  supplierId  String?                // 仕入れ先ID
  supplier    Supplier?
}
```

#### Order（新規）
```prisma
model Order {
  id           String      @id
  orderNumber  String      @unique      // 発注番号
  vesselId     String                   // 発注元船舶
  totalAmount  Int                      // 合計金額
  status       OrderStatus              // DRAFT/CONFIRMED/SHIPPED/DELIVERED
  deliveryDate DateTime?                // 希望納品日
  items        OrderItem[]              // 発注明細
}

enum OrderStatus {
  DRAFT      // カート状態
  CONFIRMED  // 発注確定
  SHIPPED    // 出荷済み
  DELIVERED  // 納品完了
  CANCELLED  // キャンセル
}
```

#### OrderItem（新規）
```prisma
model OrderItem {
  id            String
  orderId       String
  ingredientId  String
  quantity      Float         // 発注数量
  snapshotPrice Int           // 発注時点の単価
  subtotal      Int           // 小計
  isBonus       Boolean       // Boost食材フラグ（スナップショット）
}
```

#### Supplier（新規）
```prisma
model Supplier {
  id          String
  name        String
  code        String   @unique
  isActive    Boolean
  ingredients Ingredient[]
}
```

## リスク分析

### ビジネスリスク
| リスク | 対策 |
|--------|------|
| 仕入先確保 | 既存商社との提携、段階的拡大 |
| 物流構築 | 3PL活用、港湾ごとの段階展開 |
| 既存商流との競合 | 価格優位性とBoost食材で差別化 |

### 技術リスク
| リスク | 対策 |
|--------|------|
| 発注システム障害 | フォールバック手段（CSV/電話）確保 |
| スケーラビリティ | クラウドネイティブ設計 |
