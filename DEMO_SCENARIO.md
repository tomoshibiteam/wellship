## 1. 司厨シナリオ（約10分）

### 目的
司厨長が日々の献立作成と調達業務を効率化できることを体験

### 手順

1. **ログイン**
   - `chef@demo.wellship.jp` / `demo1234`
   - → `/planning` に遷移

2. **AI献立生成**
   - 「AI献立生成」ボタンをクリック
   - 条件設定（予算、アレルギー除外など）
   - 生成された週間献立を確認
   - 「献立を適用」で保存

3. **レシピ代替**
   - 特定のレシピをクリック
   - 「代替を検索」ボタン
   - 理由を選択（食材切れ、アレルギーなど）
   - 代替案から選択して置き換え

4. **調達リスト確認**
   - 左メニューから「調達リスト」へ
   - 週間必要食材リストを確認
   - PDFエクスポート（デモ）

5. **フィードバック確認**
   - 「フィードバックサマリー」へ
   - 昨日の喫食結果を確認
   - コメントを参考に次回献立に反映

### 確認ポイント
- [ ] AI生成が5秒以内に完了するか
- [ ] 代替レシピが適切に提案されるか
- [ ] 調達リストが正確に計算されるか

## 2. 船員シナリオ（約3分）

### 目的
船員が食事後30秒以内でフィードバックできることを体験

### 手順

1. **キオスクモード開始**
   - 司厨アカウントで「記録モード開始」ボタン
   - → キオスク画面に遷移

2. **NFCタッチ（デモ）**
   - 「手入力（デモ用）」をクリック
   - カードコード入力: `CREW-SAKURA-001`
   - 「確認」ボタン

3. **フィードバック入力**
   - 満足度: ★4つをタップ
   - 量: 「ちょうど良い」
   - 残し: 「完食」
   - （任意）コメント入力
   - 「送信する」ボタン

4. **完了確認**
   - サンキュー画面表示
   - 5秒後に自動リセット

### 確認ポイント
- [ ] 操作が直感的か（説明なしで使えるか）
- [ ] 入力が30秒以内に完了するか

## 3. 管理者シナリオ（約5分）

### 目的
管理者が複数船舶のフィードバックを一元管理できることを体験

### 手順

1. **ログイン**
   - `manager@demo.wellship.jp` / `demo1234`
   - → `/manager/dashboard` に遷移

2. **ダッシュボード確認**
   - 担当船舶一覧を確認
   - 直近のフィードバックサマリーを確認

3. **クルー管理**
   - 左メニューから「クルー管理」へ
   - 船員一覧を確認
   - カードコード管理

### 確認ポイント
- [ ] 複数船舶のデータが見やすく表示されるか
- [ ] 問題のあるフィードバックを素早く発見できるか

## 4. 経営層シナリオ（約5分）

### 目的
経営層がKPIを一覧で把握しESG報告に活用できることを体験

### 手順

1. **ログイン**
   - `exec@demo.wellship.jp` / `demo1234`
   - → 経営サマリーへ

2. **KPI確認**
   - 導入船舶数
   - 平均満足度
   - 完食率
   - 直近7日間の推移

3. **ESGインパクト確認**
   - フードロス削減量
   - 船員満足度スコア
   - コンプライアンス達成率

4. **レポート出力（デモ）**
   - 「ESGレポート (PDF)」ボタン
   - 「データエクスポート (CSV)」ボタン

### 確認ポイント
- [ ] 経営判断に必要な指標が揃っているか
- [ ] 投資対効果（ROI）が分かりやすいか
